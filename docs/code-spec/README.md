# Code-Spec 基线（以代码事实为准）

本目录是新的规范基线，用于替代“先写理想 spec、再追代码”的方式。

目标：
- 先固化当前真实实现（后端 + 前端）。
- 明确哪些设计应保留，哪些只作为历史兼容记录。
- 作为后续“从 spec 反推重构代码”的唯一对照输入。

判定原则：
- `保留`：实现稳定、语义清晰、对业务有长期价值。
- `观察`：实现存在但有漂移/老化，先记录，不作为硬约束。
- `剔除`：实现质量差或已明显过时，不进入新规范正文。

优先级（你当前要求：后端优先）：
1. `backend/`：主基线（Authoritative）。
2. `frontend/`：现状记录（Secondary），用于识别 SPA 与后端契约偏差。

说明：
- 本目录刻意不追求“理想形态”，只记录可验证事实与可保留设计。
- 原 `docs/spec/` 可继续保留用于历史对比，但不再视为最终真相。
